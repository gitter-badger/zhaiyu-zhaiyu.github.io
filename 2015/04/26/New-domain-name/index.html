New domain name

Apr 26 2015

As they told me, if you wanna start sth., you’d better pay for it.

And you will not fail it for the money you have paid.

Reasonable.

So I buy a domain online and resolve it to my blog on github.

I am going to renew my blog once a week, I guess. But who knows? Perhaps someday I get busy suddenly and stop it.

Well, since I have done the same thing twice this days, anyway I am kind of ‘expert’ as you said. So I would show you, in this post, how to buy a domain and bind it with you blog. It can be found any where, but, it’s more brief here.

  • First thing first, you build a blog on a site like github. If you are a guy from China mainland, you may prefer gitcafe. I mean, due to the Internet management in China P. R., you may think it’s more safe to build it on a native website. But if you are a guy like me, who believe there should be no boundary in the world, you may choice a more mature website like github. That’s your freedom.

  • To build a blog, you can choose a framework. I am using hexo. I think it’s easy to use. But if you are even more lasier than me (Oh! I cannot believe it!), tumblr is a good choice. As I know it give the user the freedom to choice your own domain.

  • Then, buy a domain. I bought one on 万网, if you like to know. And it’s OK if you buy from another firm.

  • Add a resolve record. Set type as CNAME, domain as zhaiyu-zhaiyu.github.io for me, remember to change the domain to your own blog. If yours is on github as well, it should be username.github.io. Keep the other term as default.

  • Last thing, add a file named CNAME, in which there is an only line record the domain you bought before.

Tada! Test your new domain. If it does not work, check the details. (If you really do not know the things on the github side, you’re welcome to see how I make it on my blog repo.)
